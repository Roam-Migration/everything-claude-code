**Critical bugs fixed during build:**

| # | Symptom | Cause | Fix |
|---|---------|-------|-----|
| 1 | `databases.query is not a function` | `@notionhq/client` v5 renamed to `dataSources.query` | Cast to `any`, use `dataSources.query({ data_source_id })` |
| 2 | All teams `active: false` after sync | PostgREST `.not('col', 'in', '(\'id1\',\'id2\')')` ‚Äî single-quoted UUIDs never match bare UUID column values | `(${ids.join(',')})` ‚Äî no quotes |
| 3 | `roles: 0` | `RoleRow` used `name` but Supabase column is `title` | Changed to `title` throughout |
| 4 | `people: 0` | 16 seed people with `notion_id=NULL` caused email UNIQUE conflict | Pre-delete `WHERE employment_status='legacy' AND notion_id IS NULL` |
| 5 | `people_roles` empty | Missing UNIQUE constraint on `(person_id, role_id)` | `ALTER TABLE ‚Ä¶ ADD CONSTRAINT ‚Ä¶ UNIQUE (person_id, role_id)` via Management API |
| 6 | `role_title: null` in view | Sync inserted `is_primary: false` | Changed to `is_primary: true` |
| 7 | View empty despite data | Notion stores `"Active"` (capital A); view compared `= 'active'` | `LOWER(p.employment_status) = 'active'` |
| 8 | Names like `"@Jackson"` | v5 SDK privacy-masked pages return `@Name` mention-format titles | Strip `@` prefix, use email-derived last name |
