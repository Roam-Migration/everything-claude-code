## Technical Patterns Learned

### Skill Trigger Mechanism
Custom SKILL.md files don't auto-trigger like plugin skills. CLAUDE.md trigger rules with "without waiting to be asked" are the correct way to make skills fire automatically based on context. The trigger table format works well: `| Trigger condition | Skill path |`.

### Skill Structure That Works
Phase-gated workflows > flat checklists. Each phase needs:
- A clear gate condition before proceeding
- Error recovery table (symptom → cause → fix)
- Anti-patterns with reasons (not just "don't do X" but "X causes Y")
- Key IDs/references embedded so the skill is self-contained
